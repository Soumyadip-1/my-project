 EXPERIMENT 16 — Bank Account Transfer System

 Project Overview
Project Type: Node.js REST API
Technologies Used: Express.js, MongoDB, Mongoose
Objective: Build a secure and reliable money transfer system with strong balance validation.

------------------------------------------------------------

 Core Features
- Unique user account creation and management.
- Secure money transfer between different accounts.
- Balance check with prevention of insufficient funds.
- Rollback mechanism for failed transactions.
- Robust error handling for all operations.

------------------------------------------------------------

 API Endpoints
GET /api/health → Health check endpoint
POST /api/users → Create a new user account
GET /api/users → Retrieve all user accounts
POST /api/transfer → Transfer money between accounts
GET /api/balance/:accountNumber → Check account balance by account number

------------------------------------------------------------

 Implementation Details
- Transfers follow a sequential debit → credit process.
- Automatic rollback if any transaction step fails.
- Strict input validation for account numbers and amounts.
- Prevents same-account or negative-amount transfers.

------------------------------------------------------------

 Testing & Validation
- Includes a comprehensive test suite with sample accounts.
- Tests successful transfers and insufficient balance cases.
- Verifies proper error handling for invalid inputs and accounts.

------------------------------------------------------------
 Security Measures
- Ensures sufficient balance before initiating transfers.
- Verifies account existence before transactions.
- Input sanitization and data validation throughout.
- Maintains data consistency using rollback logic.

------------------------------------------------------------

 Project Files
server.js → Main API server entry point
models/User.js → User schema and account model
test-api.js → Automated test cases for API validation
Bank-Transfer-API.postman_collection.json → Postman collection for manual testing

------------------------------------------------------------

 How to Use
1. Start the MongoDB service.
2. Run the project using: npm run dev
3. Access the server at: http://localhost:3000
4. Run automated tests using: npm test

------------------------------------------------------------

 Summary
This experiment showcases how to manage financial transactions securely using Node.js and MongoDB — without relying on database-level transactions.
It applies application-level validation and rollback mechanisms to ensure all money transfers remain safe, reliable, and consistent.
